# 工学院大学マジシャンズ・ソサエティ公式サイト

## 1. サービスを一言で表す説明
学生マジックサークルの活動実績・募集情報・公演依頼窓口をワンストップで確認できるシングルオーナー制作の静的ウェブサイトです。

## 2. サイト概要とURL
| 項目 | 内容 |
| --- | --- |
| 制作者 | 工学院大学情報学部所属のマジシャンTN（公式サイトのリニューアルを担当） |
| ページ構成 | `index.html`（ホーム） / `about.html`（活動紹介） / `contact.html`（公演依頼） |
| 想定ユーザー | 新入生、地域イベント主催者、採用担当者（短時間で活動実績と対応フローを知りたい人） |
| 現状の公開状況 | 2025-12-16 時点ではホスティング準備中。完成後はサークル公式サイトとして公開予定。 |
| 画面デモ | `images/hero-image.png` と `assets/videos/landing-intro.mp4` で主要ビジュアルを確認可能 |

## 3. 制作背景と目的
- 旧サイトはデザインが古く、依頼者視点で魅力が伝わりづらいと感じたことがリニューアルの出発点でした。
- 情報学部に在籍しているものの本格的なサイト制作は未経験だったため、「まずは一つ形にする」ことを目標に、基礎力を養いながら部活の魅力を体系的にまとめられる題材として公式サイトを選択しました。
- 新入生からは「活動曜日・場所・未経験歓迎か」を、依頼者からは「連絡時に必要な情報」をよく質問されていたので、トップとCONTACTにそれぞれ明示し、確認や聞き直しの手間を減らすことを目的にしています。

## 4. デザインやUIで意識した点
- 依頼につながりやすく信頼感を与えるため、シンプルなレイアウトと余白設計で視認性を高めつつ、モダンなガラスモーフィズム表現をアクセントにしました。
- マジックショーの照明を意識したダークテーマを既定としつつ、閲覧環境次第でライトテーマへ切り替えられるトグルを設置。黒を基調にアクセントカラーを差し込み、マジック感のある世界観を再現しています。
- ファーストビューでは動画（`landing-intro.mp4`）を初回のみ自動再生し、驚きを演出。`sessionStorage` で既視聴を判定し、2回目以降は瞬時にコンテンツが表示されるUXとしました。
- Stickyヘッダーに透過＋`backdrop-filter` を適用し、スクロール中もナビゲーションとSNS導線を常時露出。外部リンクのクリック率を高めています。
- 日本語の長い団体名が崩れないよう `text-wrap: balance` / `line-break: strict` / `white-space: nowrap` を併用して禁則処理を最適化しました。
- IntersectionObserverによるスクロールアニメーションは `prefers-reduced-motion` 設定を尊重するフォールバックを用意し、視覚演出とアクセシビリティの両立を図りました。
- ページ数を3枚に絞り、カードやCTA導線を厳選することで、初見でも迷わず必要情報へ辿り着けるシンプルさを優先しました。
- ABOUTページには活動内容が一目で分かる写真ギャラリーを配置し、イベント実績を視覚的に示して信頼感を補強しています。

## 5. 実装した主な機能と工夫
| 機能 | 実装箇所 | 工夫・理由 |
| --- | --- | --- |
| テーマ切替と永続化 | `assets/script.js` / `assets/style.css` | `localStorage` と `data-theme` で全要素を一括制御。画像も `data-src-dark/white` で同期し、`aria-pressed` を更新して支援技術に状態を伝えます。 |
| 初回限定フルスクリーン動画 | `assets/main.js` | `sessionStorage` で再生済み判定し、1セッション1回のみ再生。`ended` / `error` イベントで確実にオーバーレイを除去します。 |
| ナビ/フッターの自動更新 | `assets/script.js` | `data-page` 由来で `aria-current` を付与、フッター年号を動的更新し、メンテナンス負荷を下げました。 |
| 依頼テンプレのワンクリックコピー | `contact.html` / `assets/script.js` | `[data-copy-target]` ボタンからクリップボードへ即コピーでき、聞き取り漏れを削減します。 |
| ギャラリーのキャプション化 | `about.html` / `assets/script.js` | 画像を動的に `<figure>` へラップし、`alt` テキストを `figcaption` に転用。HTMLを簡潔に保ちながら意味付けを追加。 |
| スクロールアニメーション基盤 | `assets/scroll.css` / `assets/script.js` | `data-animate` をJSで付与し、IntersectionObserverで入場を監視。CSSで遅延やトランスフォームを制御し、JSロジックを最小化。 |

## 6. 使用技術とその役割
| 技術 | 役割 | 採用理由 |
| --- | --- | --- |
| HTML5 | 3ページの静的構成 | 未経験からDOMとセマンティクスを正しく学ぶため。環境依存なく配布可能。 |
| CSS3（Custom Properties / Flexbox / Grid） | テーマ切替、レイアウト、アニメーション | フレームワークに頼らず設計力を鍛える狙い。変数化でダーク/ライト両テーマを一元管理。 |
| Vanilla JavaScript | テーマ制御、初回動画、IntersectionObserver、クリップボード | 依存ライブラリなしでブラウザAPIを理解するために純JSを選択。 |
| 画像・動画アセット | 実績紹介とブランド印象の強化 | 実際の公演写真と動画を使って第三者の信頼感を高める。ファイル最適化は今後の課題として残す。 |

## 7. 苦労した点と学んだこと
- **テーマ切替時のFOUC対策**: 読み込み直後にライトテーマが一瞬表示される問題を、インラインスクリプトで `<html>` へ即時 `data-theme` を設定することで解決し、Critical Rendering Path の重要性を学びました。
- **動画オーバーレイのフェイルセーフ**: `ended` が発火しない例外を考慮し `error` とタイムアウトで確実にオーバーレイを除去。例外系を軽視しない姿勢を獲得しました。
- **アクセシビリティ配慮**: IntersectionObserver と `prefers-reduced-motion` の両方を扱うことで、演出とユーザーの快適さを両立させる実装パターンを理解しました。

## 8. フォルダ構成や設計の考え方
```
kogakuin_magic/
├─ index.html / about.html / contact.html   # 役割ごとにHTMLを分離
├─ assets/
│  ├─ style.css / scroll.css                # 基本スタイルと演出スタイルを分割
│  ├─ script.js / main.js                   # 汎用ロジックと初回演出を分割
│  ├─ images/ , icons/ , videos/            # メディア種別で管理し差し替えを容易に
├─ images/                                  # ファビコン・背景など共通資産
└─ LICENSE
```
- CSSを2枚に分けレンダリング順序を明示。
- JSは常時必要な機能とホーム限定演出で責務を分割し、不要なロードを避けました。
- 日本語ファイル名の実績写真を `assets/images` に集約し、コード側で扱いやすい命名としています。
